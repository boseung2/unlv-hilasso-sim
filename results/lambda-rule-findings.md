# λ 선택 규칙 비교 — val-min(현재) vs val-1se vs F1-oracle

> 실행: 2026-07-07, `compare_lambda_rules.py` (10회 반복, 프로토콜은 노트북 §1~7과 동일).
> 산출물: `lambda_rule_comparison.csv`(per-rep), `lambda_rule_summary.csv`(평균).
> 검증: val-min 결과가 notebooks/01·02의 §12 표와 소수점 4자리까지 전부 일치.

## 질문

Dataset III·IV에서 F1이 논문 대비 크게 낮은 것(LASSO III ratio 0.40)이
"held-out val MSE 최소 λ의 과다선택" 때문인가? 그렇다면 1se 규칙으로 고치면 논문에 도달하는가?

## 세 규칙

| 규칙 | 정의 | 성격 |
|---|---|---|
| val-min | val MSE 최소 λ | 현재 노트북 방식 |
| val-1se | 최소 MSE + 1·SE 이내의 가장 큰 λ (SE = std(val 제곱오차)/√n_val) | 해결 후보 |
| f1-oracle | 경로상 F1 최대 λ (참 β 사용) | 진단용 상한 — 방법 아님 |

## 발견 3가지

**1. 과다선택 진단 확증.** val-min은 III에서 평균 76~78개를 선택(참 비영 50개, 그마저 상당수 불일치)해
precision 0.12~0.15. val-1se는 선택을 7~12개로 줄여 precision 0.78~0.99로 회복.
F1 병목은 recall이 아니라 precision이었다 — 가설 그대로.

**2. val-1se는 F1을 크게 올리지만(III +48%, IV +26%) III에서 논문에는 못 미침(0.19 vs 0.32).**
대신 RME·RMSE가 논문과 거의 일치하게 됨: LASSO III 1se의 (RME_All, RMSE) = (1.741, 4.889) vs
논문 (1.790, 5.076) — ratio 0.97/0.96. val-min에서는 RME가 논문보다 뚜렷이 낮았다(0.82).
→ 논문의 (RME, F1) 조합은 val-min보다 val-1se 쪽 해와 훨씬 잘 맞는다.
논문이 실제로는 더 희소한 λ를 골랐을 가능성이 높다.

**3. (핵심) LASSO는 oracle조차 논문 F1에 못 미침 — 남은 격차는 선택 규칙 문제가 아니다.**
LASSO III oracle = 0.235 < 논문 0.321, IV oracle = 0.265 < 논문 0.293.
즉 우리 데이터에서는 λ 경로 위 어디에도 논문 수준 F1의 해가 없다.
유력 용의자는 **SNR**: 우리 III·IV의 이론 SNR = βᵀΣβ/σ² ≈ 4.4 (β·Σ·σ가 II와 동일하므로)인데
논문 보고값은 III=14.4, IV=9.5로 2~3배 높다. I·II는 SNR이 논문과 비슷했고(2.9/4.4 vs 2.9/4.2)
F1도 맞았다. 논문의 β draw가 우리 seed(34)보다 신호가 훨씬 강했다는 뜻 —
N(0,4)에서 50개를 뽑을 때 βᵀΣβ는 draw에 따라 크게 변동한다(기존 관찰: 40개 seed에서 SNR 2~75).

## 구현 주의 (버그 회고)

rpy2에서 `np.asarray(R객체)`는 R 메모리의 **뷰**일 수 있다. `del fit` + `ro.r("gc()")` 후에
그 뷰를 읽으면 해제된 메모리(전부 0 등)를 읽는 간헐 버그 발생 — 실제로 ENet IV rep5에서
bhat이 전부 0이 되어 F1 평균이 0.1909→0.1655로 왜곡됐었다.
경로 데이터를 보관할 때는 반드시 `np.array(..., copy=True)`로 소유권 있는 사본을 만들 것.
(노트북은 필요한 열만 즉시 `.copy()`하므로 안전.)
